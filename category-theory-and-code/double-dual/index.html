Double Dual 二重双対

The concept of the double dual in vector spaces is an important example that catalyzed the birth of the term “natural transformation.”

ベクトル空間における二重双対という概念は、自然変換という言葉が生まれるきっかけともなった重要な例です。

What It Means for a Transformation to Be Natural

変換が自然であるとは

In mathematics and programming, countless operations transform one object into another. However, most are merely convenient conversions. Among them, transformations that don’t depend on implementation details—such as a particular coordinate system—and arise inevitably from the structure itself are called “natural.”

数学やプログラミングにおいて、ある対象を別の形に変換する操作は無数にあります。しかし、その多くは便宜上の変換に過ぎません。その中で、実装の詳細（例えば特定の座標系）に一切依存せず、構造そのものから必然的に導き出される変換を、「自然」（Natural）と呼びます。

A prime example is the map from a vector space VVV to its double dual space V∗∗V^{**}V∗∗.

その代表例が、ベクトル空間 VVV からその二重双対空間 V∗∗V^{**}V∗∗ への写像です。

Three Layers

3つの階層

This discussion involves three layers:

  • VVV (vector space): A collection of raw data. Example: points in 3D space (x,y,z)(x, y, z)(x,y,z).

  • V∗V^*V∗ (dual space / space of measurement devices): A collection of linear functions ϕ\phiϕ that take a vector and return a numerical value. Examples:

    • When light shines from above, reading only the xxx component of the shadow cast on the ground (xyxyxy plane): ϕ(v)=x\phi(v) = xϕ(v)=x

    • Multiplying three components by their respective unit prices and calculating the total: ϕ(x,y,z)=100x+50y+10z\phi(x, y, z) = 100x + 50y + 10zϕ(x,y,z)=100x+50y+10z

  • V∗∗V^{**}V∗∗ (double dual space / space of devices): A collection of meta-functions that take a measurement device ϕ\phiϕ and return a numerical value.

この話には3つの階層が登場します。

  • VVV（ベクトル空間）:

    生身のデータの集まり。例：3次元空間の点 (x,y,z)(x, y, z)(x,y,z)。

  • V∗V^*V∗（双対空間 / 測定器の空間）:

    ベクトルを受け取って数値を返す線形関数 ϕ\phiϕ の集まり。例：

    • 光が上から当たっているとき、地面（xyxyxy平面）に落ちた影の xxx 成分だけを読み取る ϕ(v)=x\phi(v) = xϕ(v)=x

    • 3つの成分の重さに、それぞれの単価を掛けて合計金額を出す ϕ(x,y,z)=100x+50y+10z\phi(x, y, z) = 100x + 50y + 10zϕ(x,y,z)=100x+50y+10z

  • V∗∗V^{**}V∗∗（二重双対空間 / 装置の空間）

    測定器 ϕ\phiϕを受け取って数値を返すメタ関数の集まり。

In category theory, within the category of vector spaces Vect\mathbf{Vect}Vect:

  • Object VVV: The world where raw data (vectors vvv) lives.

  • Dual functor (−)∗:Vect→Vectop(-)^: \mathbf{Vect} \to \mathbf{Vect}^{op}(−)∗:Vect→Vectop: Maps an object VVV to the set of morphisms from VVV to scalar values (V∗V^V∗).

  • Morphism ϕ∈V∗\phi \in V^*ϕ∈V∗: In category theory, this is the morphism V→RV \to \mathbb{R}V→R itself.

  • Double dual functor (−)∗∗:Vect→Vect∗∗(-)^{}: \mathbf{Vect} \to \mathbf{Vect}^{}(−)∗∗:Vect→Vect∗∗: Maps object VVV to V∗∗V^{}V∗∗ and morphism fff to f∗∗f^{}f∗∗.

圏論の言葉で書くならベクトル空間の圏 Vect\mathbf{Vect}Vect において、

  • 対象 VVV: 生身のデータ（ベクトル vvv）が住む世界

  • 双対関手 (−)∗:Vect→Vectop(-)^: \mathbf{Vect} \to \mathbf{Vect}^{op}(−)∗:Vect→Vectop:
    対象 VVV を「VVV からのスカラー値への射の集合（V∗V^    V∗）」へ移す操作。

  • 射 ϕ∈V∗\phi \in V^*ϕ∈V∗: 圏論では V→RV \to \mathbb{R}V→R という射そのものです。

  • 二重双対関手(−)∗∗:Vect→Vect(-)^{}: \mathbf{Vect} \to \mathbf{Vect}(−)∗∗:Vect→Vect** : 対象 VVV を V∗∗V^{}V∗∗ へ、射 fff を f∗∗f^{**}f∗∗ へ移す。

Unnatural Dual V∗V^*V∗

不自然な双対 V∗V^*V∗

For a vector space VVV (a collection of data), its dual space V∗V^*V∗ is the collection of linear functions (measurement devices) that take a vector and return a numerical value.

VVV (data): Points in 3D space (x,y,z)(x, y, z)(x,y,z)

V∗V^*V∗ (dual / measurement devices): “Calculation rules” such as measuring shadow length or computing total cost

ベクトル空間 VVV（データの集まり）に対して、その双対空間 V∗V^*V∗ とは、ベクトルを受け取って数値を返す線形関数（測定器）の集まりです。

VVV（データ）: 3次元空間の点 (x,y,z)(x, y, z)(x,y,z)

V∗V^*V∗（双対 / 測定器）: 影の長さを測る、合計金額を出すといった「計算ルール」

A vector space VVV and its dual V∗V^*V∗ are structurally equivalent (isomorphic) when they have the same dimension. Any linear function ϕ\phiϕ that takes a vector v=(x,y,z)v = (x, y, z)v=(x,y,z) in 3D space and returns a numerical value must take the following form:

ベクトル空間 VVV とその双対 V∗V^*V∗ は、同じ次元であれば構造的に等しい（同型）です。3次元空間のベクトル v=(x,y,z)v = (x, y, z)v=(x,y,z) を受け取って数値を返す線形関数 ϕ\phiϕ は、必ず次の形になります。

ϕ(x,y,z)=ax+by+cz\phi(x, y, z) = ax + by + czϕ(x,y,z)=ax+by+cz

To specify this “measurement rule,” we need three parameters (coefficients): a,b,ca, b, ca,b,c. In other words, the “variations of linear functions” themselves form a 3D space (V∗V^*V∗), representable by coordinates (a,b,c)(a, b, c)(a,b,c).

この「測定ルール」を特定するには、a,b,ca, b, ca,b,c という3つのパラメータ（係数）を決める必要があります。つまり、「線形関数のバリエーション」そのものが、(a,b,c)(a, b, c)(a,b,c) という座標で表せる3次元の空間（V∗V^*V∗）を作っているのです。

Why the Dual Is Unnatural

なぜ双対は不自然なのか

VVV (data) and V∗V^*V∗ (measurement devices) are both 3D, but there’s no natural rule for transforming between them that everyone can agree on.

VVV（データ）も3次元、V∗V^*V∗（測定器）も3次元ですが、その間の変換には誰にでも納得がいくような自然なルールが存在しません。

For example, what if we directly map the point v=(1,2,3)v = (1, 2, 3)v=(1,2,3) to the function ϕ=(1,2,3)\phi = (1, 2, 3)ϕ=(1,2,3) (ϕ(v)=1x+2y+3z\phi(v) = 1x + 2y + 3zϕ(v)=1x+2y+3z)? As you can see from the components and unit prices example, this is completely meaningless.

例えば、v=(1,2,3)v = (1, 2, 3)v=(1,2,3) という点を、ϕ=(1,2,3)\phi = (1, 2, 3)ϕ=(1,2,3) という関数（ϕ(v)=1x+2y+3z\phi(v) = 1x + 2y + 3zϕ(v)=1x+2y+3z）にそのまま対応させたらどうでしょう。これは成分と単価の例を考えてもわかるように全く無意味です。

In the shadow example, the basis in space is inherently arbitrary. We can choose completely different bases for the same space—such as Earth-based vs Sun-based coordinates, world coordinates vs camera coordinates, or centimeter units vs inch units. The shadow length is represented by different parameters of ϕ\phiϕ depending on the chosen basis (This argument applies even when not dealing with physical space).

また、影の例について考えると、空間における基底はそもそも恣意的なものです。同じ空間に対して全く異なる基底（例えば地球基準に対して太陽基準、ワールド座標に対してカメラ座標、センチメートル単位に対してインチ単位など）を取ることができ、影の長さは基底によって異なる ϕ\phiϕ のパラメータで表されます（この議論は物理的な空間でなくても同様に当てはまります）。

The Natural Double Dual V∗∗V^{**}V∗∗

自然な二重双対 V∗∗V^{**}V∗∗

In the double dual, we add one more layer: we treat a vector vvv as a device that takes a function ϕ\phiϕ and executes it by substituting vvv into it.

二重双対では更に1つレイヤーを加えて、ベクトル vvv を、受け取った関数 ϕ\phiϕ に自分 vvv を代入して実行する装置とみなします。

Start with a specific vector v=(3,4,5)v = (3, 4, 5)v=(3,4,5). No matter what computational rule ϕ\phiϕ (measurement device) is presented, you can substitute your value into ϕ\phiϕ and return the result.

  • If a total-cost-computing ϕ2\phi_2ϕ2​ comes, compute and return ϕ2(3,4,5)\phi_2(3, 4, 5)ϕ2​(3,4,5).

  • If a shadow-measuring ϕ1\phi_1ϕ1​ comes, compute and return ϕ1(3,4,5)\phi_1(3, 4, 5)ϕ1​(3,4,5).

特定のベクトル v=(3,4,5)v = (3, 4, 5)v=(3,4,5) をスタート地点にします。目の前に、どんな計算ルール ϕ\phiϕ（測定器）が差し出されても、あなたはその ϕ\phiϕ に自分の値を代入して結果を答えることができます。

• 影を測る ϕ1\phi_1ϕ1​ が来れば ϕ1(3,4,5)\phi_1(3, 4, 5)ϕ1​(3,4,5) を計算して答える。
• 合計金額を出す ϕ2\phi_2ϕ2​ が来れば ϕ2(3,4,5)\phi_2(3, 4, 5)ϕ2​(3,4,5) を計算して答える。

This concept—executing ϕ\phiϕ on vvv—becomes an object in V∗∗V^{**}V∗∗. The transformation from vector to device can be written as:

この vvv に対して ϕ\phiϕ を実行するという概念そのものが、V∗∗V^{**}V∗∗ における対象になります。ベクトルから装置への変換を式で書くと以下のようになります。

v↦evv(where evv(ϕ)=ϕ(v))v \mapsto \text{ev}_v \quad (\text{where } \text{ev}_v(\phi) = \phi(v))v↦evv​(where evv​(ϕ)=ϕ(v))

v↦evv(where evv(ϕ)=ϕ(v))v \mapsto \text{ev}_v \quad (\text{where } \text{ev}_v(\phi) = \phi(v))v↦evv​(where evv​(ϕ)=ϕ(v))

This correspondence αV:V→V∗∗\alpha_V: V \to V^{**}αV​:V→V∗∗ is a mapping built into the structure itself, unchanged no matter what coordinate system you choose.

この対応 αV:V→V∗∗\alpha_V: V \to V^{**}αV​:V→V∗∗ は、座標系をどう選ぼうが、構造そのものに組み込まれたマッピングです。

Let’s write it in code as well.

コードでも書いてみましょう。

/**
 * Dual Space (V*): 
 * The function is the "Active" part. It consumes the data.
 */
// phi: V -> R
const shadowX = (v: Vector3): number => v.x;

const result = shadowX(v); // phi(v)

/**
 * Double Dual Space (V**): 
 * The data is wrapped into a "Device". 
 * It now consumes the function.
 */

// alpha: V -> V**
// This function transforms a raw vector into a "Double Dual" device.
const toDevice = (v: Vector3) => {
    // This returned function is the element of V**
    return <R>(phi: (data: Vector3) => R): R => {
        return phi(v);
    };
};

const ev_v = toDevice(v); // ev_v is an element of V**
const result = ev_v(shadowX); // The device "runs" the measurement.

コードの例からこれはカリー化だと気づいた方もいるでしょう。二重双対の装置（評価写像）は、「適用」という演算をカリー化したものと言い換えることができます。

Conditions in Category Theory

圏論における条件

This mapping α\alphaα satisfies the naturality condition (commutativity) in category theory.

  • Route 1: Transform vector vvv by linear transformation fff, then move it into V∗∗V^{**}V∗∗.

  • Route 2: Move vector vvv into V∗∗V^{}V∗∗ first, then transform it there (by f∗∗f^{}f∗∗).

このマッピング α\alphaα は圏論に於ける自然性の条件（可換性）を満たすことができます。

  • Route 1: ベクトル vvv を線形変換 fff で加工してから、V∗∗V^{**}V∗∗ の世界へ移す。

  • Route 2: ベクトル vvv を先に V∗∗V^{}V∗∗ の世界へ移してから、その世界で加工（f∗∗f^{}f∗∗）する。

That these two routes coincide for all fff proves this transformation preserves only structure—the relationship of substitution—independent of specific numerical values.

この2つが全ての fff に対して一致することは、この変換が中身の具体的な数値によらず、構造（代入という関係性）だけを保存していることを証明しています。
